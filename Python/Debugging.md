# Debugging
## Errors
- ZeroDivisionError: Returns when attempting a division operation on a denominator of zero
- SyntaxError: Returns when misspellings, mismatched punctuations, or incorrect operant present
- NameError: Returns when an variable occurs before assignment, incorrect spelling, or undefined
- Indentation Error: Returns when the content of a for or while loop, function is not nested correctly
- TypeError: Returns when trying to use an incompatible data type to perform a task
- ValueError: Returns when a value provided is not within an acceptable range
- StopIteration: Returns when trying to iterate over an iterator that has no more items left

## Error Management
- Errors occur when trying to execute code that violates a rule within Python
- Python errors are also referred to as exceptions
- The name exception is used interchangeably with errors
- Errors within Python code will cause programs to terminate
- When writing code it is important to anticipate where error may occur and try to prevent it
- Preemptively anticipating errors in code can create more efficient and targeted debugging
- Errors are displayed in the terminal indicating where an execution failure occurred in the script
- Error outputs return the line of code where the error was detected using a carrot indicator
- Error outputs will return a traceback which includes what type of error occurred and where
- Error indicators appears under the portion of code where the error was detected
